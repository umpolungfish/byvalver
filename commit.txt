fix: Implement proper ML prediction tracking and metrics recording

## Summary
This commit fixes the critical issue where ML strategist showed "Predictions Made: 0"
despite actively using neural network inference for strategy selection. The ML model was
performing predictions and learning from feedback, but the prediction count was never
incremented because predictions were not being recorded in the metrics system.

## Problem Analysis

### Observed Symptoms
```
=== ML STRATEGIST PERFORMANCE SUMMARY ===
Instructions Processed: 3702
Strategies Applied: 3693

--- Model Performance ---
Predictions Made: 0          ← ISSUE: No predictions recorded
Current Accuracy: 0.00%
Avg Prediction Confidence: 0.0000

--- Learning Progress ---
Learning Enabled: YES
Total Feedback Iterations: 7395  ← Learning WAS happening
Positive Feedback: 3693
```

### Root Cause
The `ml_reprioritize_strategies()` function in `src/ml_strategist.c` was:
1. ✅ Extracting instruction features
2. ✅ Running neural network forward pass
3. ✅ Scoring and reordering strategies based on ML confidence
4. ❌ **Never calling `ml_metrics_record_prediction()`**

The ML model was actively being used for strategy selection, but the metrics tracking
system had no visibility into which strategies were predicted or whether predictions
were correct.

## Solution Implemented

### Three-Component Prediction Tracking System

#### Component 1: Prediction State Variables (src/ml_strategist.c:39-41)
```c
static strategy_t* g_last_predicted_strategy = NULL;
static double g_last_prediction_confidence = 0.0;
```
Added global state to track the ML model's prediction between strategy selection
and outcome resolution.

#### Component 2: Prediction Storage (src/ml_strategist.c:398-407)
Modified `ml_reprioritize_strategies()` to store predictions:
- After reordering strategies by ML confidence scores
- Store top-ranked strategy (index 0) as the prediction
- Store ML confidence score for that strategy
- Prediction held in pending state until outcome is known

```c
if (*strategy_count > 0) {
    g_last_predicted_strategy = applicable_strategies[0];
    g_last_prediction_confidence = scores_copy[0];
}
```

#### Component 3: Outcome-Based Recording (src/ml_strategist.c:537-562)
Modified `ml_provide_feedback()` to verify and record predictions:
- Compare applied strategy against stored prediction
- Determine correctness:
  - Correct: predicted strategy was used AND succeeded
  - Incorrect: different strategy used OR predicted strategy failed
- Call `ml_metrics_record_prediction()` with actual outcome
- Clear prediction state for next instruction

```c
if (g_ml_metrics && g_last_predicted_strategy != NULL) {
    int prediction_correct = 0;

    if (applied_strategy != NULL && applied_strategy == g_last_predicted_strategy) {
        prediction_correct = success ? 1 : 0;
    } else if (applied_strategy == NULL && g_last_predicted_strategy != NULL) {
        prediction_correct = 0;
    } else if (applied_strategy != NULL && applied_strategy != g_last_predicted_strategy) {
        prediction_correct = 0;
    }

    ml_metrics_record_prediction(g_ml_metrics, prediction_correct, g_last_prediction_confidence);
    g_last_predicted_strategy = NULL;
    g_last_prediction_confidence = 0.0;
}
```

## Metrics Tracking Flow

```
┌─────────────────────────────────────────────────────────────┐
│ Instruction with null bytes detected                        │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ get_strategies_for_instruction()                            │
│ - Collect applicable strategies                             │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ ml_reprioritize_strategies()                                │
│ - Extract instruction features (128-dimensional)            │
│ - Neural network forward pass (128→256→200)                 │
│ - Score strategies with NN output                           │
│ - Sort by ML confidence                                     │
│ ✨ NEW: Store top strategy + confidence                     │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ Apply strategy (generate null-free code)                    │
└─────────────────────────┬───────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│ ml_provide_feedback()                                       │
│ - Compare applied strategy vs prediction                    │
│ - Determine if prediction was correct                       │
│ ✨ NEW: Record prediction with outcome                      │
│ - Update neural network weights (learning)                  │
│ - Update metrics (feedback iterations)                      │
│ ✨ NEW: Clear prediction state                              │
└─────────────────────────────────────────────────────────────┘
```
