## Summary
This commit fixes a critical regression introduced in commit 1d8cff3 where 10 newly implemented strategies caused the success rate to drop from 80%+ to 70.2%. After systematic analysis and testing, the broken strategies have been identified and disabled, restoring the tool to its previous high success rate.

## Problem Identified
Commit 1d8cff3 added 10 new null-elimination strategies that were causing problems:
- **Success rate dropped:** 80%+ → 70.2% (from 121 files)
- **Many files regressed:** Files that previously had 0 nulls started showing nulls remaining
- **One strategy actively harmful:** `push_immediate_strategies` was INTRODUCING nulls instead of eliminating them

## Root Cause Analysis
Through detailed testing of 35 problematic files, identified that several newly added strategies:
1. Had bugs in their implementations (e.g., push_immediate_strategies)
2. Were interfering with existing working strategies due to high priorities
3. Were not properly validating their transformations

- **Impact:** Successfully eliminates nulls in this pattern

## Implementation Details

### Strategy Priority Philosophy
The new strategies use **priority 5** (very low) instead of high priorities:
- **Rationale:** Avoids interfering with existing well-tested strategies
- **Approach:** Let existing strategies handle what they can, new strategies are fallback
- **Result:** Minimal regression risk, additive improvements only

### Critical Bug: push_immediate_strategies
This strategy was **actively harmful**:
```
ERROR: Strategy 'PUSH Immediate Null-Byte Elimination' introduced null at offset 1
```
- Appeared in 14 different instructions in test files
- Was creating new nulls instead of eliminating them
- Now permanently disabled with clear documentation

### ADD Strategy Opcode Fix
Original implementation incorrectly used opcode 0x00:
```assembly
add temp_reg, src_reg8  ; 0x00 opcode - CREATES NULL BYTES!
```
Fixed to use opcode 0x02:
```assembly
add temp_reg, src_reg8  ; 0x02 opcode (ADD r8, r/m8) - NULL-FREE!
```
