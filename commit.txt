feat(strategies): Implement 5 novel bad-character elimination strategies

Add 5 new specialized transformation strategies for advanced bad character
elimination, bringing total strategy count from 143 to 148+. These strategies
target specific instruction patterns commonly found in network shellcode, flag
manipulation, and loop constructs.

IMPLEMENTED STRATEGIES:

1. Strategy 24: BSWAP Endianness Transformation (Priority 85)
   - File: src/bswap_endianness_transformation_strategies.c/.h
   - Detects MOV with immediate values containing bad characters
   - Checks if byte-swapped version has fewer bad characters
   - Generates: MOV reg, swapped_value; BSWAP reg
   - Particularly useful for:
     * IP addresses (127.0.0.1, 192.168.x.x)
     * Port numbers in socket structures
     * Network byte order conversions

2. Strategy 21: PUSHF/POPF Bit Manipulation (Priority 81)
   - File: src/pushf_popf_bit_manipulation_strategies.c/.h
   - Replaces flag-setting instructions with bad character encodings
   - Uses PUSHF/POPF to manipulate EFLAGS register directly
   - Targets: STC, CLC, STD, CLD, CMC instructions
   - VERIFIED: STC (F9) → PUSHF; POP EAX; OR EAX, 0x01; PUSH EAX; POPF
   - Transformation: 1 byte → 9 bytes

3. Strategy 17: BSF/BSR Bit Scanning (Priority 80)
   - File: src/bit_scanning_constant_strategies.c/.h
   - Detects power-of-2 immediate values with bad characters
   - Uses bit position calculation and shifting to reconstruct values
   - Example: MOV EAX, 0x100 → MOV EAX, 1; SHL EAX, 8
   - Applicable to:
     * Power-of-2 constants (0x1, 0x2, 0x4, ... 0x80000000)
     * Bitmask values in Windows API calls
     * Flag values in system structures

4. Strategy 22: LOOP Comprehensive Variants (Priority 79)
   - File: src/loop_comprehensive_strategies.c/.h
   - Transforms LOOP/LOOPE/LOOPNE with bad character displacements
   - LOOP target → DEC ECX; JNZ target
   - LOOPE target → DEC ECX; JZ skip; JE target; skip:
   - VERIFIED: LOOP -2 (E2 FE) → DEC ECX; JNZ -3 (49 75 FD)
   - Size impact: LOOP (2→3 bytes), LOOPE/LOOPNE (2→5 bytes)

5. Strategy 13: Atomic Operation Encoding Chains (Priority 78)
   - File: src/atomic_operation_encoding_strategies.c/.h (already existed)
   - Transforms LOCK-prefixed atomic operations with bad characters
   - Removes LOCK prefix to create non-atomic equivalents
   - WARNING: Loses atomicity - only for single-threaded shellcode
   - Supports: LOCK XADD, LOCK CMPXCHG, LOCK INC/DEC, etc.
   - Verified registration in v3.5
