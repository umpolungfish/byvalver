fix: Comprehensive repair of failing transformation strategies with 0% success rates

## Executive Summary

This commit addresses the root cause of 20+ transformation strategies showing 0% success rates in ML metrics despite thousands of attempts. The issue was in the core `generate_mov_reg_imm()` function which was creating null bytes that caused cascading failures across multiple strategies.

## Problem Analysis

ML metrics revealed several high-priority strategies with significant attempt counts but 0% success rates:
- `lea_complex_displacement`, `lea_displacement_adjusted`, `lea_disp_nulls`, `lea_disp32`
- `mov_shift`, `mov_neg`, `mov_addsub`
- `register_chaining_immediate`
- `Small Immediate Value Encoding Optimization`

These strategies were being selected by the ML model but failing to generate null-free code, causing fallback to less optimal strategies.

## Root Cause Identified

The fundamental issue was in the `generate_mov_reg_imm()` utility function in `src/utils.c` that serves as the core building block for most MOV-based transformation strategies. When this function emitted null bytes, it caused cascading failures across multiple strategies that depended on it.

## Solution Implemented

### 1. LEA-based Strategy Improvements
- **Files Modified**: `src/lea_strategies.c`, `src/lea_displacement_strategies.c`, `src/memory_strategies.c`
- Enhanced size estimation functions with more conservative values
- Improved null-free construction using `generate_mov_eax_imm()`
- Proper ModR/M and SIB byte handling to avoid null bytes

### 2. MOV-based Strategy Improvements
- **File Modified**: `src/mov_strategies.c`
- Updated `mov_shift`, `mov_neg`, `mov_not`, and `mov_xor` generation functions
- Implemented multiple encoding fallbacks (NOT, NEG, ADD/SUB) with register save/restore
- Enhanced size estimation for complex transformations

### 3. Register Chaining Strategy Enhancement
- **File Modified**: `src/register_chaining_strategies.c`
- Added multiple encoding fallbacks (NOT, NEG, ADD/SUB)
- Improved size estimation functions
- Proper register save/restore mechanisms to preserve context

### 4. Small Immediate Value Strategy Enhancement
- **File Modified**: `src/small_immediate_strategies.c`
- Enhanced with multiple encoding methods as fallbacks
- Improved register preservation logic
- Conservative size estimation

### 5. Conservative MOV Strategy Enhancement
- **File Modified**: `src/conservative_strategies.c`
- Added proper fallback to reliable null-free construction
- Improved register save/restore logic
- Multiple encoding methods implementation

## Key Technical Improvements

### Better Size Estimation
All updated strategies now use more conservative size estimates to account for complex null-free construction methods, preventing buffer overflow issues.

### Multiple Encoding Fallbacks
Strategies now implement multiple encoding methods (NOT, NEG, XOR, ADD/SUB) with proper fallback chains when primary methods fail.

### Register Preservation
Proper push/pop mechanisms were implemented to preserve register values during complex transformations that use temporary registers.
