CRITICAL FIX: Disable broken strategies that introduce nulls instead of eliminating them

## Summary

This commit addresses critical bugs in BYVALVER where several null-elimination strategies were actually introducing null bytes instead of eliminating them. The fix involves disabling the problematic strategies until they can be properly rewritten. This significantly improves the success rate of null-elimination across test samples.

## Problem

BYVALVER had critical bugs in several strategies that were causing it to fail on most files in the BIG_SAMPLER test set:
1. `mov_register_remap` strategy was appending original instruction with nulls instead of transforming
2. `contextual_register_swap` strategy was introducing nulls instead of eliminating them
3. `hash_based_resolution` strategy had buggy implementation causing nulls to be introduced
4. `enhanced_peb_traversal` strategy had incomplete implementation causing failures

These bugs caused BYVALVER to output shellcode with null bytes despite the processing, defeating the entire purpose of the tool.

## Solution

### Disabled Broken Strategies:
- **mov_register_remap** - In `src/register_allocation_strategies.c`, set `can_handle` to return 0
- **contextual_register_swap** - In `src/register_allocation_strategies.c`, set `can_handle` to return 0  
- **hash_based_resolution** - In `src/peb_api_resolution_strategies.c`, set `can_handle` to return 0
- **enhanced_peb_traversal** - In `src/peb_api_resolution_strategies.c`, set `can_handle` to return 0
- **peb_conditional_jumps** - In `src/peb_api_resolution_strategies.c`, set `can_handle` to return 0

### Implementation Changes:
- Modified `can_handle_*` functions to return 0 (disabled) instead of processing instructions
- Left existing function implementations intact for future reference and potential fixes
- Maintained all other working strategies unchanged

## Technical Implementation

### Before:
```c
// Buggy implementation that introduces nulls
int can_handle_mov_register_remap(cs_insn *insn) {
    // Check for register operations with nulls
    if (insn->id == X86_INS_MOV && /* has nulls */) {
        return 1;  // Handle this - BUT introduces nulls!
    }
    return 0;
}
```

### After:
```c
// Disabled to prevent null introduction
int can_handle_mov_register_remap(cs_insn *insn) {
    // Disable this broken strategy - it introduces nulls instead of eliminating them
    (void)insn;
    return 0;
}
```
