feat: Add Agent Menagerie — AI-powered auto-technique generation pipeline

Introduces a four-stage autonomous agent pipeline (agents/) built on the
AjintK multi-provider LLM framework. A single command discovers all
existing strategies, proposes a novel one, generates a complete C
implementation, patches strategy_registry.c, and verifies the build.

Changes:
========

1. Agent Pipeline (agents/)
   - agents/__init__.py — package init
   - agents/strategy_discovery_agent.py — scans src/, catalogs 340+
     strategies across all categories, produces LLM-summarised gap
     analysis for downstream agents
   - agents/technique_proposal_agent.py — given the catalog, asks the
     LLM to propose one genuinely novel technique not already covered;
     returns structured JSON (name, approach, arch, priority, example)
   - agents/code_generation_agent.py — generates complete .h + .c
     files conforming to strategy_t using strategy.h / utils.h /
     mov_strategies.c as live reference material
   - agents/implementation_agent.py — writes generated files to src/,
     applies three targeted patches to strategy_registry.c (include,
     forward declaration, register call), runs make

2. Entry Point
   - run_technique_generator.py — orchestrates the four-stage pipeline;
     supports --dry-run, --arch, --provider, --model, --verbose
   - Multi-provider: anthropic (default), deepseek, qwen, mistral, google
   - Provider API key validated before any LLM calls are made

3. Documentation
   - README.md — added ## AGENT MENAGERIE section with pipeline table,
     quick-start, options reference, requirements; added nav link and
     TOC entry; added docs/AGENT_MENAGERIE.md to documentation table
   - docs/USAGE.md — added ## What's New in v4.3 section with full
     usage guide, option table, agent architecture table, registry
     patching details, example run output, and provider reference table
