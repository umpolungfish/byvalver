## New Features

### Core TUI Implementation
- **9 Main Menu Options** covering all CLI functionality:
  1. Process Single File - Real-time single file processing with visual feedback
  2. Batch Process Directory - Multi-file processing with live progress tracking
  3. Configure Processing Options - Toggle biphasic, PIC, ML, verbose, dry-run modes
  4. Set Bad Characters - Manual hex entry or profile selection (13 predefined profiles)
  5. Output Format Settings - Choose from 5 formats (raw, C, Python, PowerShell, hexstring)
  6. ML Metrics Configuration - ML strategy selection and metrics tracking
  7. Advanced Options - XOR encoding, timeouts, limits, validation settings
  8. Load/Save Configuration - INI-style configuration file management
  9. About byvalver - Version and help information

### Visual File Browser (src/tui/tui_file_browser.h/.c)
- **Directory navigation** with arrow keys or vi-style j/k keys
- **File/directory distinction** with [FILE] and [DIR] visual indicators
- **Human-readable file sizes** (B, KB, MB, GB formatting)
- **Extension filtering** support (e.g., *.bin pattern matching)
- **Intelligent path handling** - Auto-navigates to parent directory if file path provided
- **Sorted display** - Directories first, then alphabetical ordering
- **Three browser modes**:
  - BROWSER_MODE_SELECT_FILE: Navigate directories, select files only
  - BROWSER_MODE_SELECT_DIRECTORY: Select directories for batch processing
  - BROWSER_MODE_SELECT_BOTH: Flexible selection of either type
- **Keyboard controls**:
  - ENTER: Select file or navigate into directory
  - SPACE: Select current directory (in directory mode)
  - q: Cancel and return to menu

### Batch Processing with Live Updates (src/tui/tui_screens.c)
- **Real-time progress bar** with filled/empty indicators (e.g., [========    ] 52/100)
- **Live statistics** with color-coded status:
  - ✅ Successful (green - COLOR_PAIR(4))
  - ❌ Failed (red - COLOR_PAIR(3))
  - ⏭️ Skipped (normal text)
- **Current file display** in bold text showing file being processed
- **Next file preview** in yellow/dim text showing upcoming file
- **Dynamic strategy statistics table** with real-time updates:
  - Top 10 most-used strategies displayed
  - Success/failure counts per strategy
  - Success rate percentages calculated live
  - Color-coded by performance (green ≥80%, yellow 50-79%, red <50%)
  - Updates every 50ms during processing
- **Final summary screen** with:
  - Total files processed count
  - Success/failure/skipped counts with percentages
  - Total input/output bytes tracked
  - Average size ratio calculation
  - Press-any-key to return to main menu

### Configuration File Management (src/cli.c)
- **INI-style format** with 6 sections:
  - [general]: verbose, quiet, show_stats, validate_output, dry_run
  - [processing]: use_biphasic, use_pic_generation, encode_shellcode, xor_key,
                  strategy_limit, timeout_seconds, max_size
  - [output]: output_format (raw|c|python|powershell|hexstring)
  - [bad_characters]: bad_chars (comma-separated hex values)
  - [ml]: use_ml_strategist, metrics_enabled, metrics_export_json/csv,
          metrics_show_live, metrics_output_file
  - [batch]: file_pattern, recursive, preserve_structure, continue_on_error
- **Full parser implementation** in load_config_file() (cli.c:595-710)
- **Example configuration** provided in example.conf
- **Load/Save functionality** integrated into TUI menu system

### Real Processing Integration (src/processing.h - NEW FILE)
- **Exposed process_single_file()** to TUI via new header
- **Made function non-static** in src/main.c for TUI access
- **Real strategy initialization**: init_strategies(), init_obfuscation_strategies()
- **Actual bad-character context**: init_bad_char_context() with bitmap
- **Real file discovery**: find_files() for batch processing
- **Actual batch statistics**: batch_stats_t tracking with real strategy usage
- **No simulated progress** - all metrics are from actual processing

### Stdout/Stderr Redirection (src/tui/tui_screens.c)
- **File descriptor management** using dup(), dup2() to redirect stdout/stderr
- **Redirect to /dev/null** during batch processing to prevent console corruption
- **Clean ncurses display** maintained without verbose debug output interference
- **Restore file descriptors** after processing for summary display
- **Screen stability** - only progress UI visible during processing

                                strategy_usage_t → strategy_stats_t

### File Browser Issues
- **Fixed directory selection bug**: Browser now only navigates into directories in FILE mode
  instead of incorrectly selecting them as files
- **Fixed path validation**: Added stat() checks to handle file/directory/invalid paths correctly
- **Fixed "Cannot read directory 'output.bin'" error**: Browser validates path type before opening
- **Intelligent parent navigation**: When given a file path, browser navigates to parent directory
  instead of attempting to open file as directory

### Processing Issues
- **Removed ALL mock code**: Eliminated simulated progress, placeholder statistics,
                             fake processing loops
- **Fixed console output corruption**: Stdout/stderr redirection prevents verbose logs
                                       from overwriting ncurses display
- **Fixed "appears to hang" on large directories**: Added progress messages during initialization:
  - "Scanning directory... Found X files"
  - "Initializing strategies..."
  - "Initializing bad character context..."
  - "Setting up batch statistics..."
  - "Starting batch processing..."
